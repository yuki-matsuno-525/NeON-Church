## 仕様決定済み事項

### ブックマーク表記（Phase 15）

内部実装・DB・API は `bookmark` のままで、UI表記のみ「お気に入り」にする。マイグレーション不要。

初期タグ（10件）: 信仰 / 祈り / 愛 / 倫理 / 律法 / 預言 / 歴史 / 奇跡 / 教え / 疑問

実装: Render PostgreSQL はカスタム拡張（pg_bigm・pgroonga）不可の可能性が高いため、まず `icontains` で実装し、データ量増加後に Meilisearch へ移行する。

## Phase 13 — コメントシステムバグ修正 ✅ 完了（2026-05-06）

**優先度: 最高。後続フェーズのUI改善の前提となるバグを先に直す。**

### 修正対象

| # | 内容 | 種別 |
|---|------|------|
| 3 | 章コメント・書コメントの投稿が「投稿先の節が見つかりません」エラーになる | バックエンドバグ |
| 7 | 返信への返信（depth ≥ 2）が表示されない | フロントエンドバグ |
| 8 | ページ下部の章コメント欄に節コメントが混入している。節・章・書コメントを完全に分離する | フロント/バックエンドバグ |

### 設計方針

- コメントの `target_type`（verse / chapter / book）をバックエンドで正しく判別できているか確認
- フロントのコメント取得クエリが `verse_id` / `chapter_id` / `book_id` を適切に使い分けているか確認
- ツリー再帰レンダリングがすべての depth で動くように修正

成果物: 章・書へのコメント投稿成功、返信ツリーが正しく表示される、節/章/書コメントが独立して表示される
