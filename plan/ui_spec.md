# NeON Church フロントエンド UI 仕様

> 実装前の合意ドキュメント。変更が生じた場合はこのファイルを更新する。

---

## 1. 確定した設計方針

| 項目 | 決定事項 |
|------|---------|
| テーマ | ミニマル（A） |
| フォント | Noto Sans JP |
| ダークモード | `next-themes` による切り替え（ライト・ダーク両対応） |
| サイドバー | アコーディオン、固定幅 200px |
| 節コメント | 右パネル（節選択時に展開） |
| 章コメント | 読書ページ下部 |
| 書コメント | 書ページ（章グリッド + コメント） |
| 検索 | MVP 対象外 |

## 4. レイアウト構造

```
┌─────────────────────────────────────────────────────┐
│ Navbar (56px)                                       │
├──────────┬──────────────────────────┬───────────────┤
│          │                          │               │
│ Sidebar  │  メインコンテンツ        │ CommentPanel  │

│          │                          │ ※節選択時のみ │

- CommentPanel は読書ページで節が選択されたときのみ表示。他ページでは非表示。
- モバイル（< 768px）では Sidebar をハンバーガーメニューに変更（MVP では後回し可）。

## 5. 共通コンポーネント

### Navbar

```
[NeON Church]  ............  [ブックマーク] [🔔²] [アバター▾] [☀/🌙]
```

- ロゴクリック → `/matthew/1`
- 🔔 未読バッジ: 未読通知数を表示。0件は非表示。
- アバターは未実装（MVP ではクリック不可でよい）
- 未ログイン時: 右端に「ログイン」ボタンのみ

### Sidebar（アコーディオン）

```
マタイ  ▾
  第1章  ← アクティブ時 accent 背景
  第2章
  第3章
マルコ  ▸
ルカ    ▸
ヨハネ  ▸
```

- 書名クリック → 書ページ（`/[book]`）へ遷移 & アコーディオン展開
- 章クリック → 読書ページ（`/[book]/[chapter]`）へ遷移
- 複数書を同時展開可（排他制御なし）

     ▲ 5  返信

**右パネル（CommentPanel）:**

```

「アブラハムの子であるダビデの子…」
──────────────────────
[コメント入力欄     ]
[             投稿 ]

- 節を再クリックまたは他の節をクリックで切り替え
- パネルを閉じるボタン（×）を右上に配置
- 未ログイン状態では入力欄を非表示、「ログインしてコメントする」リンクを表示

### 6-2. 書ページ `/[book]`

```
マタイによる福音書

章を選択
┌───┬───┬───┬───┐
│ 1 │ 2 │ 3 │ 4 │  ← ホバーで accent tint
│ 5 │ 6 │ 7 │   │
└───┴───┴───┴───┘

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
この書へのコメント (5)

[コメント入力欄              ]
[                   投稿する ]

```

### 6-3. ブックマーク一覧 `/bookmarks`

- カード形式。書名・章:節 + 本文抜粋。
- カードクリックで読書ページへ遷移（該当節を選択状態で開く）。
- 未ログインでアクセス → ログインページへリダイレクト。

### 6-4. 通知一覧 `/notifications`

- 未読: 左に accent ボーダー + accent tint 背景。
- 返信通知 → クリックで該当コメントのある節を右パネルで開く（MVP では読書ページに飛ぶだけで可）。
- 「すべて既読」ボタン。
- 未ログインでアクセス → ログインページへリダイレクト。

### 6-5. ログイン `/login`

- ユーザー名 + パスワードのシンプルなフォーム。
- 「新規登録はこちら」→ `/register`。
- 成功後 → 直前のページ（または `/matthew/1`）へリダイレクト。

### 6-6. 新規登録 `/register`

- ユーザー名 + メール + パスワードのフォーム。
- 成功後 → `/matthew/1` へリダイレクト。

## 7. 認証フロー

- `next-themes` の隣に認証状態を管理する Context（`AuthContext`）を用意。
- アプリ起動時に `GET /api/auth/me/` を叩いてログイン状態を確認。
  - 200 → ユーザー情報を Context に保存。
  - 401 → 未ログイン状態として扱う。
- Cookie は HTTP-only のため JS から直接読めない。`/me` の結果のみを信頼する。
- ログアウト → `POST /api/auth/logout/` → Context クリア → ルートへリダイレクト。
