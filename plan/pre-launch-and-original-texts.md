# 公開前メモ：追加機能の優先順位 と 原文テキスト導入

HN / Reddit で一気に流入する前提での「やるべきこと」整理。

---

## 1. すでにある（追加不要）

- レート制限（投稿 10/分・ログイン 5/分・通報 5/分）
- 通報 + 管理者モデレーション（論理削除）
- Google ログイン
- 利用規約 / プライバシー / コミュニティガイドライン
- SEO（sitemap / robots / OGP）
- Sentry（エラー監視）

守りはおおむね固まっている。

---

## 2. 公開前に埋めたい穴（効く順）

### 優先度：高

**(1) Render の“スリープ”対策 ― 最重要の技術リスク**
- 無料プランはアクセスが無いとサーバーが寝て、復帰に約 50 秒。
- バズの最初の訪問者が 50 秒待ち or エラー → 初動を取りこぼす。
- 対策：launch 前後だけでも有料プラン(Starter)に上げる／10 分おきに `/health` を叩く cron(keep-alive)。

**(2) パスワード再設定（「お忘れですか？」）が無い**
- メール登録者がパスワードを忘れると詰む。
- 最低限：ログイン画面で Google ログインを主役にし、メール登録を控えめに。
- 本格対応：再設定フロー実装（メール送信設定が前提）。

**(3) アクセス解析が無い**
- 今は誰が何を見たか不明＝バズの効果測定ができない。
- Plausible 等の軽量・プライバシー配慮型を導入（読者層にも合う）。

### 優先度：中（第一印象・離脱対策）

**(4) “過疎に見える”問題**
- コメント 0 だと初見に「誰もいない」と映り離脱。
- 有名な数節（創世記1章・ヨハネ1章・エノク書冒頭など）に質の高いコメント/Q&A を数件シード。

**(5) ログイン無しで読める導線の明確化**
- 読むだけなら登録不要、をトップで明示（「まず読む → 気が向いたらログインしてコメント」）。

**(6) 節ごとの共有リンク + OGP**
- SNS で節を貼られたとき、節単位 URL ときれいなプレビューが出ると拡散に乗りやすい。実装済みなら確認のみ。

**推奨着手順：(1) → (3) → (4)**

## 3. 原文テキストの導入（聖書・外典・偽書）

### 結論
- **ヘブライ語聖書・ギリシャ語新約・ギリシャ語LXX（第二正典含む）は、パブリックドメイン版で現実的に導入可能。**
- **偽書（エノク書など）の原語は入手性・文字対応の壁が高く、当面は翻訳のままが現実的。**
- コンセプト（テキストと解釈・断片が交差するデータベース／対訳ビュー構想）に原文追加は強く合致する。

### コンセプト・設計上の相性
- 既存の「対訳ビュー（複数翻訳を並べて読む）」構想とそのまま噛み合う。原文も“1つの版(translation)”として追加できる（`books.ts` の translations 配列 + `import_scripture`）。
- エノク書は既に「翻訳プロジェクトの元テキスト」。原語があれば共同翻訳がさらに強化される。

### 入手・ライセンス（重要）

| 区分 | 推奨版 | ライセンス | 備考 |
|---|---|---|---|
| ヘブライ語聖書 | Westminster Leningrad Codex (WLC) | PD / オープン | Sefaria・Tanach.us 等から機械可読データ入手可 |
| ギリシャ語新約 | Textus Receptus(Scrivener 1894) / Westcott-Hort 1881 / Tischendorf 8th | PD | いずれも公有 |
| ギリシャ語新約(別案) | SBLGNT | 無料だが帰属表示要 | 商用・有償配布に条件。表示すれば利用可 |
| ギリシャ語新約 | NA28 / UBS5 | 著作権あり(独聖書協会) | **不可** |
| 七十人訳(LXX) | Swete (1909) | PD | **第二正典(トビト記・知恵の書 等)も LXX に含む** → 外典の原語化に有効 |
| 七十人訳(別案) | Rahlfs | 権利配慮が必要 | 慎重に |
| 偽書(エノク等) | ゲエズ語(エチオピア語)中心＋アラム語/ギリシャ語断片 | 版による | クリーンな機械可読データが乏しく難易度高 |

### 技術的な課題
1. **ヘブライ語の RTL（右→左）表示**対応が必要。
2. **章節番号(versification)のズレ**：MT / LXX / 各訳で節の区切り・番号が異なる → 節アラインメントの工夫が要る。
3. **ライセンス明記の運用**：各原文版の出典・ライセンスを Licenses ページと各文献ページに記載。

### 推奨ロードマップ
1. まず PD で堅い構成：**ヘブライ語 WLC** ＋ **ギリシャ語新約(PD版)** を「原文」版として追加。
2. **第二正典を LXX(Swete) で原文化**（外典の一部を原語で読める）。
3. **偽書の原語は将来課題**（断片・特殊文字対応）。当面は翻訳＋共同翻訳で。
