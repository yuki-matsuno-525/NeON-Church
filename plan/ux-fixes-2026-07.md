## 方針（ユーザー決定事項）

- **語彙**: 「正典 / 外典 / 偽書（偽典）」という語をカテゴリ名・説明文に使わない。
  ただし「周縁に置かれた文書に光を当てる」のように**歴史的事実に言及する文脈では使ってよい**。
  格付けとして使うのが禁止であって、歴史を語る語としては禁止ではない。
- **検索**: UI 言語と検索対象の訳を切り離す。
- **翻訳プロジェクトのライセンス**: 最も緩いもの（CC0 1.0 / パブリックドメイン相当）に統一。

### 1. 検索と UI 言語の切り離し

- 原因は `backend/bible/views.py:22` の `LANG_TRANSLATIONS`。`lang=en` だと KJV 等に絞られ「神」が構造上 0 件になる。
- 対応: API は `lang` の代わりに `translation`（複数可）を受ける。未指定なら全訳を対象にする。
  フロントは検索ページで訳を選べるようにし、UI 言語は初期値を決めるだけにする。
- 既存バグも同時に修正: `views.py:24` の `"L. S. A. W ells (EN)"`（スペース混入）。
