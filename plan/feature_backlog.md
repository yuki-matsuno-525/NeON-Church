# 機能バックログ（未実装アイデア集）

NeON Church の未実装アイデアを、優先度と実装可能性の 2 軸で整理したもの。`plan/ui_ux_release_plan.md`（UX-1〜14 で完了）の後続として、次に着手する機能を判断するための一覧。

## 評価軸

### 優先度
- **★★★**：プロダクトの差別化に直結、β公開やリリースに不可欠
- **★★**：価値はあるが必須ではない、時間をかけて取り組む
- **★**：長期構想、将来検討

### 可能性（実装難度・リソース確保のしやすさ）
- **◎**：標準的な技術で実装可能、新規依存少ない
- **○**：要リサーチだが見通しが立つ
- **△**：ライセンス確認・コンテンツ確保が前提
- **×**：大規模、別アプリ規模

## 推奨順序

short-term から long-term への自然な流れ：

1. **A-1 OAuth ログイン** — β公開時の登録ハードル下げ、技術的に標準
2. **B-1 章・節構造のない文献のデータモデル設計** — 外典・教父文献の前提
3. **B-2 外典・偽典の収録** — プロジェクトのアイデンティティ、最大の差別化
4. **B-3 earlychristianwritings.com 等の文献追加** — 上記 B-1/B-2 を活かして拡張
5. **A-6 キリスト教カレンダー** — 軽量で UI 上の差別化
6. **C-5 地図機能** — 読書体験を深める
7. その他

各項目に着手する際は、本ファイルの該当行を「進捗」表として更新する想定（Phase 名や着手日を追記）。
