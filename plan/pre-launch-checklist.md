## 優先度付きタスクリスト

### P0: ブロッカー（公開前に必ず直す）

- [ ] **OAuth 動作確認**
  - Google / GitHub ログインのエンドツーエンドテスト（本番環境または staging）
  - 実装はあるが未テスト状態（`fix/lint` が最新コミット）
  - 壊れていると SNS シェアでのサインアップ率ゼロになる

- [ ] **デモ・ゲストユーザー体験の確認**
  - ログインなしでどこまで閲覧・体験できるか確認
  - シードデータ（15 ユーザー・200+ コメント）が本番 DB に入っているか
  - HN 読者は登録を求められると即離脱する

- [ ] **全テストのグリーン確認**
  - `pytest`（backend）/ `npm test`（frontend）を実行してパスを確認
  - E2E（`npm run e2e`）の主要フローが通ること

### P1: 第一印象（公開前日までに完了）

- [ ] **ホームページのファーストビュー**
  - ログアウト状態で「何のサービスか」5秒で伝わるか確認
  - ヒーローテキスト・キャッチコピーが明確か
  - CTA ボタン（「読む」「登録」）が目立つか

- [ ] **About ページの品質**
  - 外典・偽書も正典と同等に扱うコアコンセプトが伝わるか
  - HN ポスト文の説明と About ページの内容が一致しているか

- [ ] **モバイル最終確認**
  - 主要ページ（/, /read, /[book]/[chapter], /about）をスマホ幅で確認
  - Navbar の折りたたみが正常動作するか

- [ ] **SEO / OGP**
  - og:title, og:description, og:image が設定されているか
  - Twitter Card（SNS シェア時のプレビュー）が機能するか
  - sitemap.ts / robots.ts が正しく機能しているか

### P2: 品質担保（公開後 1 週間以内に対応）

- [ ] **パフォーマンス計測**
  - Lighthouse でホームページ・章ページのスコア確認
  - Core Web Vitals（LCP, CLS, FID）が緑か

- [ ] **エラー状態・ローディング状態の確認**
  - ErrorState コンポーネントが全ページで使われているか
  - ネットワーク遅延時に Skeleton / Spinner が表示されるか
  - 存在しない URL で 404 が適切に出るか

- [ ] **アクセシビリティ**
  - キーボードのみで主要導線（読む → コメント投稿）を完走できるか

### P3: 継続改善（公開後のフィードバックを見てから）

- [ ] **外典・偽書のコンテンツ追加**
  - Coptic SCRIPTORIUM のインポートが完了しているか確認

- [ ] **フィードバック収集の導線**
  - `/feedback` ページが機能しているか
  - GitHub Issues への誘導が適切か

- [ ] **リリースノート作成**
  - GitHub Releases で Release 1-4 の内容を公開

## Reddit/HN ポスト時の注意

### タイトル案
- HN: `Show HN: NeON Church – Read the Bible (incl. Apocrypha) with Reddit-style comments`
- Reddit（r/programming / r/selfhosted）: 同様のトーン

### ポストに含めるべき内容
1. サービス URL + デモできる節の URL（例: /genesis/1）
2. 外典・偽書を正典と同等に扱うコンセプトの説明
3. 技術スタック（Next.js + Django + PostgreSQL）
4. オープンソースかどうかの明記（GitHub URL）
5. フィードバックを歓迎する一言
